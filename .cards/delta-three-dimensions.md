Workflow 的本质不是"自动化脚本"，而是**团队协作经验的沉淀容器**。它在三个维度上持续迭代，形成可度量的增量（Delta）：

**知识 Delta**：领域知识、规则、判断标准的积累。载体是 Skill、Frontmatter Schema、Eval Rubric。每次 Suspend 点的经验蒸馏都增加知识 Delta。

**能力 Delta**：AI 处理复杂任务的能力提升。载体是 Agent Adapter、工具链、模型选择。随着 Workflow 对 Agent 提出更高要求，能力边界被持续推动。

**流程 Delta**：Workflow 本身的拓扑迭代。载体是 Graph 编排、Suspend 节点增减、角色定义。流程不是静态的，它随着实践反馈不断进化。

三者持续沉淀，Workflow 越用越"聪明"——不是 AI 本身变聪明，而是**团队的集体智慧被结构化地固化进了系统**。这与 [[three-learning-carriers]] 的 Memory/Skill/Workflow 三层载体形成映射：知识 Delta 对应 Memory 和 Skill，能力 Delta 对应 Agent，流程 Delta 对应 Workflow 本身。

参见 [[five-phase-iteration-loop]] 了解三重 Delta 是如何被系统性地产出的。
